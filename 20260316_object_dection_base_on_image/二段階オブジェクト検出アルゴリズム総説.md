## 1. 二段階オブジェクト検出アルゴリズムの概要

### 1.1 二段階オブジェクト検出とは？

二段階（Two-Stage）オブジェクト検出アルゴリズムとは、2つの段階を経てオブジェクト検出タスクを完了するアルゴリズムです：

**第1段階**：候補領域の生成（Region Proposals）
- 元の画像からオブジェクトを含む可能性のある領域を抽出
- Region Proposal Network (RPN) または Selective Search とも呼ばれる

**第2段階**：候補領域の分類と位置推定
- 各候補領域から特徴を抽出
- オブジェクト分類（何の物体か）
- バウンディングボックス回帰（物体の位置を正確に特定）

### 1.2 二段階アルゴリズムの特徴

**メリット**：
- 検出精度が高い、特に小さいオブジェクトの検出に優れる
- 候補領域生成段階で良好な事前情報を提供
- 位置推定が正確

**デメリット**：
- 推論速度が遅い
- 計算リソースの消費が大きい
- リアルタイム性の要求を満たすのが難しい

### 1.3 典型的な応用シーン
- 速度よりも精度が重視されるアプリケーション
- 医用画像診断
- 衛星画像分析
- 自動運転のバックエンド検証

## 2. R-CNN (Regions with CNN features)

### 2.1 アルゴリズムの思想

R-CNN はRoss Girshickらが2014年に提案した、ディープラーニングをオブジェクト検出分野に導入した画期的な研究です。その核心的な思想は以下の通りです：

1. **領域抽出**：Selective Searchアルゴリズムを用いて画像から約2000個の候補領域を抽出
2. **特徴抽出**：各候補領域を固定サイズ（227×227）にリサイズし、畳み込みニューラルネットワーク（CNN）で特徴を抽出
3. **分類と回帰**：
   - 各領域の分類にSVMを使用
   - 線形回帰器でバウンディングボックスを修正

### 2.2 モデル構造

```
入力画像 (任意サイズ)
    ↓
選択的探索 (Selective Search)
    ↓
約2000個の候補領域
    ↓
画像リサイズ (227×227)
    ↓
CNN特徴抽出 (AlexNet/VGG)
    ↓
特徴ベクトル (4096-d)
    ↓
┌─────────────┬─────────────┐
│   SVM分類   │ ボックス回帰│
└─────────────┴─────────────┘
```

### 2.3 トレーニングプロセス

**ステップ1：CNNの事前学習**
- ImageNetでCNN（AlexNetなど）を事前学習

**ステップ2：CNNのファインチューニング**
- オブジェクト検出データセットでCNNをファインチューニング
- 最終層をN+1クラス（N個のオブジェクトクラス + 背景）に置換

**ステップ3：SVMのトレーニング**
- 各候補領域のCNN特徴を抽出
- 各クラスごとに独立したSVM分類器をトレーニング

**ステップ4：ボックス回帰器のトレーニング**
- 候補ボックスの位置を修正するための線形回帰モデルをトレーニング

### 2.4 主なメリット

1. **画期的な貢献**
   - 初めてCNNをオブジェクト検出に適用
   - 検出精度を大幅に向上（mAPを約30%から53%へ）

2. **高い検出精度**
   - CNNが抽出する特徴は強力な表現力を持つ
   - Selective Searchが生成する候補領域の品質が高い

3. **柔軟性**
   - 異なるCNNバックボーンに置換可能
   - 様々なデータセットに適用可能

### 2.5 主なデメリット

1. **極めて遅い速度**
   - 1枚の画像を処理するのに約47秒（GPU上）
   - 原因：各候補領域ごとに独立してCNNの順伝播を行う必要がある

2. **複雑なトレーニングプロセス**
   - 多段階トレーニング（CNNファインチューニング、SVMトレーニング、回帰器トレーニング）
   - 大量の中間特徴を保存する必要がある
   - トレーニングプロセスの最適化が困難

3. **重複計算の問題**
   - 候補領域間に大量の重複が存在
   - 同じ領域が複数回繰り返しCNN特徴を計算される

4. **固定サイズの制限**
   - 候補領域を固定サイズにリサイズする必要がある
   - 画像の歪みや情報損失の可能性

5. **メモリ消費が大きい**
   - 大量の中間結果を保存する必要がある
   - 特徴キャッシュが大量のディスク容量を消費

### 2.6 性能指標
- **mAP (VOC 2010)**: 53.3%
- **推論時間**: ~47秒/画像（GPU）
- **バックボーン**: AlexNet/VGG-16

## 3. SPP Net (Spatial Pyramid Pooling Network)

### 3.1 アルゴリズムの思想

SPP NetはHe Kaimingらが2015年に提案したもので、核心的な革新は**空間ピラミッドプーリング層（Spatial Pyramid Pooling）**の導入です。

解決された主要な問題：
- R-CNNが入力画像を固定サイズに要求する
- 画像のリサイズによる歪みや情報損失
- 大量の重複計算

核心的な思想：
1. 画像全体に対して一度だけCNNの順伝播を実行
2. SPP層を用いて異なるサイズの特徴マップを固定長の特徴ベクトルに変換
3. 特徴マップから候補領域に対応する特徴を抽出

### 3.2 SPP層の原理

空間ピラミッドプーリング層は、任意のサイズの入力特徴マップを固定サイズの出力に変換できます。

```
特徴マップ (W × H × C)
    ↓
┌─────────────┬─────────────┬─────────────┐
│  1×1 bin    │  2×2 bins   │  4×4 bins   │
│ (最大プーリング) │ (最大プーリング) │ (最大プーリング) │
└─────────────┴─────────────┴─────────────┘
    ↓           ↓           ↓
  1×C        4×C         16×C
    ↓           ↓           ↓
    └───────────┴───────────┘
            ↓
    連結: 1×C + 4×C + 16×C = 21×C
            ↓
      固定次元特徴ベクトル
```

例：VGG-16の512チャンネル出力の場合、1×1、2×2、4×4のピラミッドを使用
- 出力次元: 1×512 + 4×512 + 16×512 = 21×512 = 10752次元

### 3.3 モデル構造

```
入力画像 (任意サイズ)
    ↓
CNN畳み込み層 (完全な順伝播を1回実行)
    ↓
特徴マップ (任意サイズ)
    ↓
┌─────────────────────────┐
│   選択的探索で候補ボックス生成 │
└─────────────────────────┘
    ↓
特徴マップから候補領域を抽出
    ↓
┌─────────────────────────┐
│      SPP層 (ピラミッドプーリング) │
└─────────────────────────┘
    ↓
固定次元特徴ベクトル
    ↓
┌─────────────┬─────────────┐
│   SVM分類   │ ボックス回帰│
└─────────────┴─────────────┘
```

### 3.4 主なメリット

1. **大幅な速度向上**
   - 画像全体に対して一度だけCNNの順伝播を実行
   - R-CNNより24-102倍高速

2. **任意の入力サイズを受け入れ可能**
   - 画像を固定サイズにリサイズする必要がない
   - 画像の元のアスペクト比と詳細を保持

3. **重複計算の問題を解決**
   - CNN特徴を一度だけ計算
   - 候補領域を特徴マップから抽出し、重複計算を回避

4. **受容野の一貫性**
   - SPP層は多スケールの特徴表現を生成可能
   - 異なるスケールの空間情報をキャプチャ

### 3.5 主なデメリット

1. **トレーニングはまだ多段階**
   - エンドツーエンドのトレーニングが不可能
   - トレーニングプロセスが複雑

2. **特徴抽出の制限**
   - 依然として最後の畳み込み特徴から抽出
   - 多層の特徴を十分に活用していない

3. **候補領域の品質への依存**
   - 依然としてSelective Searchに依存
   - 候補領域生成が遅い

### 3.6 性能指標
- **mAP (VOC 2007)**: 59.2%
- **推論時間**: ~0.5-2秒/画像
- **速度向上**: R-CNNより24-102倍高速

## 4. Fast R-CNN

### 4.1 アルゴリズムの思想

Fast R-CNNはRoss Girshickが2015年に提案したもので、R-CNNの大幅な改良です。主な革新点：

1. **共有畳み込み特徴**：画像全体に対して一度だけCNNの順伝播を実行
2. **RoI Pooling層**：異なるサイズの候補領域を固定サイズの特徴マップにマッピング
3. **マルチタスク損失**：分類とバウンディングボックス回帰を同時に最適化し、エンドツーエンドのトレーニングを実現
4. **統一トレーニング**：SVMを別個にトレーニングする必要がなくなった

### 4.2 RoI Poolingの原理

RoI (Region of Interest) Poolingは、任意サイズのRoI特徴を固定サイズ（例：7×7）にマッピングします。

```
RoI: (x, y, w, h)
特徴マップ上の対応領域: (x', y', w', h')
    ↓
k×k個のサブ領域に分割 (通常k=7)
    ↓
各サブ領域で最大値を取得
    ↓
出力: k×k×C (7×7×C)
```

SPPとの違い：
- SPPは複数のスケールのピラミッドプーリングを使用
- RoI Poolingは単一スケールのプーリングのみ（例：7×7）
- RoI Poolingはオブジェクト検出タスク専用に設計され、より効率的

### 4.3 モデル構造

```
入力画像 (任意サイズ)
    ↓
CNN畳み込み層 (完全な順伝播を1回実行)
    ↓
特徴マップ
    ↓
選択的探索で候補領域を生成 (~2000個)
    ↓
RoIを特徴マップにマッピング
    ↓
RoI Pooling (7×7×Cに統一)
    ↓
フラット化 + 全結合層
    ↓
┌─────────────┬─────────────┐
│  分類スコア  │ ボックス回帰│
│ (N+1クラス)  │ (4N個のパラメータ) │
└─────────────┴─────────────┘
```

### 4.4 マルチタスク損失関数

Fast R-CNNは結合損失関数を使用して分類と位置推定を同時に最適化：

```
L(p, u, t^u, v) = L_cls(p, u) + λ[u ≥ 1]L_loc(t^u, v)
```

ただし：
- `p`: 予測される分類確率分布（softmax出力）
- `u`: 真のクラスラベル
- `t^u`: クラスuの予測バウンディングボックスオフセット
- `v`: 真のバウンディングボックスオフセット
- `λ`: バランスパラメータ（通常λ=1）
- `[u ≥ 1]`: 指示関数（背景の場合は位置推定損失を計算しない）

**分類損失**（交差エントロピー）：
```
L_cls(p, u) = -log(p_u)
```

**位置推定損失**（Smooth L1損失）：
```
L_loc(t^u, v) = Σ smooth_L1(t_i^u - v_i)

smooth_L1(x) = {
    0.5x²           if |x| < 1
    |x| - 0.5       otherwise
}
```

### 4.5 トレーニングプロセス

**単一ステージエンドツーエンドトレーニング**：
1. SGDを用いてネットワーク全体を最適化
2. ミニバッチトレーニング（各回2枚の画像、128個のRoI）
3. 階層的サンプリング戦略を採用：32個の正サンプル + 96個の負サンプル

**正負サンプルの定義**：
- **正サンプル**: IoU ≥ 0.5
- **負サンプル**: 0.1 ≤ IoU < 0.5

### 4.6 主なメリット

1. **大幅なトレーニング速度向上**
   - 単一ステージトレーニング、SVMの事前トレーニングが不要
   - SGDによるエンドツーエンド最適化が可能
   - R-CNNより9倍高速なトレーニング

2. **推論速度向上**
   - R-CNNより213倍高速なテスト速度
   - SPP Netよりも向上

3. **より高い精度**
   - マルチタスク損失による結合最適化
   - R-CNNとSPP Netより高い精度

4. **統一されたフレームワーク**
   - 特徴をディスクに保存する必要がない
   - トレーニングと推論のプロセスが簡潔

5. **より良い特徴の活用**
   - RoI Poolingはwarpingより効果的
   - 空間情報を保持

### 4.7 主なデメリット

1. **候補領域生成が依然としてボトルネック**
   - 依然としてSelective Searchに依存
   - 約2000個の候補領域の生成に2-3秒かかる
   - 総推論時間の大部分を占める

2. **リアルタイム検出ではない**
   - 総推論時間約0.3-0.5秒/枚
   - リアルタイムアプリケーションの要求を満たすのが困難

3. **トレーニングが事前生成された候補領域に依存**
   - 候補領域を事前に生成する必要がある
   - 前処理ステップが追加される

### 4.8 性能指標
- **mAP (VOC 2007)**: 66.9% (VGG-16)
- **mAP (VOC 2012)**: 68.4% (VGG-16)
- **推論時間**: ~0.3秒/画像（GPU、候補領域生成を除く）
- **トレーニング時間**: ~9.5時間（VOC 2007+2012、VGG-16）

## 5. Faster R-CNN

### 5.1 アルゴリズムの思想

Faster R-CNNはShaoqing Renらが2015年に提案したもので、二段階検出アルゴリズムのマイルストーンです。核心的な革新：

1. **Region Proposal Network (RPN)**：ニューラルネットワークを用いてSelective Searchを置換
2. **共有畳み込み特徴**：RPNとFast R-CNNがCNN畳み込み層を共有
3. **ほぼリアルタイムの検出速度**：高精度を維持しながら大幅に速度を向上

重要なブレークスルー：**候補領域生成を完全にニューラルネットワークに統合し、真のエンドツーエンド検出を実現。**

### 5.2 RPN (Region Proposal Network)

#### 5.2.1 RPN構造

RPNは完全畳み込みネットワークで、畳み込み特徴マップ上で直接候補領域を生成します。

```
畳み込み特徴マップ (W × H × 512, VGG-16)
    ↓
3×3畳み込み (512チャンネル)
    ↓
┌─────────────────┬─────────────────┐
│  1×1畳み込み (2k) │  1×1畳み込み (4k) │
│  (オブジェクト/非オブジェクト) │  (ボックス回帰)   │
└─────────────────┴─────────────────┘
      ↓               ↓
   分類スコア        オフセット
      ↓               ↓
   約2k個のanchor per location
```

ただし`k`は各位置に事前に設定されたanchorの数（通常k=9）。

#### 5.2.2 Anchor機構

各特徴マップの位置にk個の異なるスケールとアスペクト比のanchorを事前に設定：

```
k = 9個のanchors
  - 3種類のスケール: 128², 256², 512²
  - 3種類のアスペクト比: 1:1, 1:2, 2:1
```

各位置で生成されるもの：
- **objectnessスコア**：そのanchorがオブジェクトか背景かを予測（2×9個の出力）
- **box回帰オフセット**：anchorから真のボックスへのオフセットを予測（4×9個の出力）

#### 5.2.3 RPN損失関数

```
L({pi}, {ti}) = (1/N_cls) Σ L_cls(pi, pi*) + λ(1/N_reg) Σ pi* L_reg(ti, ti*)
```

ただし：
- `pi`: anchor iがオブジェクトである確率
- `ti`: anchor iの予測されるバウンディングボックスオフセット
- `pi*`: 真のラベル（1は正サンプル、0は負サンプル）
- `ti*`: 真のバウンディングボックスオフセット
- `N_cls`, `N_reg`: 正規化パラメータ（ミニバッチサイズとanchor数）
- `λ`: バランスパラメータ（通常λ=10）

**正負サンプルの定義**：
- **正サンプル**: 
  - 真のボックスと最高のIoU
  - または任意の真のボックスとIoU > 0.7
- **負サンプル**: すべての真のボックスとIoU < 0.1

### 5.3 モデル構造

```
入力画像
    ↓
CNN畳み込み層 (共有)
    ↓
畳み込み特徴マップ
    ↓
┌──────────────────┬──────────────────┐
│      RPN         │                  │
│  (候補領域生成)   │                  │
└──────────────────┴──────────────────┘
         ↓
   RoI Proposals
    ↓
RoI Pooling
    ↓
全結合層
    ↓
┌─────────────┬─────────────┐
│  分類スコア  │ ボックス回帰│
└─────────────┴─────────────┘
```

### 5.4 トレーニング戦略

#### 5.4.1 四段階交互トレーニング

RPNとFast R-CNNが畳み込み層を共有しているため、交互にトレーニングが必要：

**ステップ1**：RPNをトレーニング
- 最初からRPNをトレーニング
- ImageNetで事前トレーニングされたCNNを使用

**ステップ2**：RPNが生成したトレーニングデータでFast R-CNNをトレーニング
- 共有畳み込み層を固定
- RPNが生成した候補領域でFast R-CNNをトレーニング

**ステップ3**：RPNをファインチューニング
- Fast R-CNNのパラメータを使用して共有畳み込み層を初期化
- RPN固有の層を固定し、共有畳み込み層をファインチューニング

**ステップ4**：Fast R-CNNをファインチューニング
- 共有畳み込み層を固定
- Fast R-CNN固有の層をファインチューニング

#### 5.4.2 近似結合トレーニング

より効率的な方法：RPNとFast R-CNNを一つのネットワークに統合し、統一されたマルチタスク損失を使用。

```
L = L_rpn + L_fast_rcnn
```

### 5.5 主なメリット

1. **大幅な速度向上**
   - 候補領域生成が2秒から0.01秒に短縮
   - 総推論時間約0.2秒/枚（VGG-16）
   - ほぼリアルタイム（5 fps）に達

2. **精度の維持または向上**
   - Fast R-CNNより高いmAP
   - 共有特徴がモデルの表現力を向上

3. **真のエンドツーエンドトレーニング**
   - 候補領域生成が完全に学習可能
   - 統一された最適化目標

4. **パラメータ共有による計算量削減**
   - RPNと検出ネットワークが畳み込み特徴を共有
   - 計算複雑さを大幅に削減

5. **高い拡張性**
   - 様々なバックボーンに適用可能
   - 他の視覚タスクに拡張可能

### 5.6 主なデメリット

1. **まだリアルタイムに達していない**
   - VGG-16上で約5 fps
   - リアルタイムアプリケーション（ビデオ監視など）にはまだ不十分

2. **高い計算リソース要件**
   - 中間特徴を保存するために大きなメモリが必要
   - ハードウェアの要求が高い

3. **小さいオブジェクト検出の制限**
   - 深い層の特徴が小さいオブジェクト情報を失う
   - マルチスケール特徴融合が必要

4. **anchor設計への感応性**
   - データセットに合わせてanchorのスケールとアスペクト比を調整する必要がある
   - 不適切なanchorは性能に影響を与える

### 5.7 性能指標
- **mAP (VOC 2007)**: 78.8% (ResNet-101)
- **mAP (VOC 2012)**: 75.9% (ResNet-101)
- **推論時間**: 
  - VGG-16: ~0.2秒/枚 (5 fps)
  - ResNet-101: ~0.14秒/枚 (7 fps)
- **RPN時間**: ~10ms/画像

## 6. アルゴリズムの比較とまとめ

### 6.1 発展の過程

```
2014: R-CNN
    ↓ (速度が遅すぎる、重複計算)
2015: SPP Net (任意サイズ入力、特徴共有)
    ↓ (多段階トレーニング)
2015: Fast R-CNN (RoI Pooling、エンドツーエンドトレーニング)
    ↓ (候補領域が依然としてボトルネック)
2015: Faster R-CNN (RPNニューラルネットワーク、ほぼリアルタイム)
```

### 6.2 性能の比較

| アルゴリズム | mAP (VOC 07) | 推論時間 | 候補領域生成 | トレーニング方式 | R-CNNに対する速度 |
|--------------|--------------|----------|--------------|------------------|-------------------|
| R-CNN | 53.3% | ~47秒 | Selective Search | 多段階 | 1× |
| SPP Net | 59.2% | ~0.5-2秒 | Selective Search | 多段階 | 24-102× |
| Fast R-CNN | 66.9% | ~0.3秒 | Selective Search | 単一段階 | 213× |
| Faster R-CNN | 78.8% | ~0.2秒 | RPN (ニューラルネットワーク) | エンドツーエンド | ~235× |

### 6.3 核心技術の進化

| 技術点 | R-CNN | SPP Net | Fast R-CNN | Faster R-CNN |
|--------|-------|---------|------------|--------------|
| 候補領域生成 | Selective Search | Selective Search | Selective Search | RPN (ニューラルネットワーク) |
| 特徴共有 | ❌ 各RoIが独立計算 | ✅ 全体で1回計算 | ✅ 全体で1回計算 | ✅ 全体で1回計算 |
| プーリング方式 | Warping | SPP (マルチスケール) | RoI Pooling (単一スケール) | RoI Pooling |
| 固定入力サイズ | ✅ リサイズが必要 | ❌ 任意サイズ | ❌ 任意サイズ | ❌ 任意サイズ |
| トレーニング方式 | 多段階 (CNN+SVM+回帰) | 多段階 | 単一段階 (マルチタスク損失) | エンドツーエンド |
| 分類器 | SVM | SVM | Softmax | Softmax |

### 6.4 各アルゴリズムのメリット・デメリットまとめ

#### R-CNN
**メリット**：
- 画期的な研究、初めてCNNをオブジェクト検出に適用
- 従来の方法より大幅に高い精度

**デメリット**：
- 極めて遅い（47秒/枚）
- 重複計算、リソースの浪費
- 複雑なトレーニング、多段階

#### SPP Net
**メリット**：
- 任意の入力サイズを受け入れ可能
- 大幅な速度向上（24-102倍）
- 重複計算の問題を解決

**デメリット**：
- 依然として多段階トレーニング
- 特徴を十分に活用できていない

#### Fast R-CNN
**メリット**：
- エンドツーエンドトレーニング、統一されたフレームワーク
- マルチタスク損失、より高い精度
- トレーニングと推論速度が大幅に向上

**デメリット**：
- 候補領域生成が依然としてボトルネック
- リアルタイムではない

#### Faster R-CNN
**メリット**：
- RPNがニューラルネットワークで候補領域を生成
- 真のエンドツーエンドトレーニング
- ほぼリアルタイム（5-7 fps）
- 最高精度

**デメリット**：
- まだリアルタイムに達していない
- 計算リソース要件が高い
- anchor設計への感応性

### 6.5 重要な技術革新点

1. **特徴共有**
   - SPP Net、Fast R-CNN、Faster R-CNNの共通特徴
   - 全体で一度特徴を計算し、重複計算を回避

2. **プーリング層の設計**
   - SPP: マルチスケールピラミッドプーリング
   - RoI Pooling: 固定サイズプーリング、検出専用設計

3. **エンドツーエンドトレーニング**
   - Fast R-CNNがマルチタスク損失を導入
   - Faster R-CNNが完全なエンドツーエンドを実現

4. **候補領域生成**
   - R-CNN/SPP/Fast: Selective Search（従来のアルゴリズム）
   - Faster: RPN（学習可能なニューラルネットワーク）

### 6.6 実際の応用の推奨

**R-CNNを選択する場合**：
- ほぼない（学習用参考としてのみ）

**Fast R-CNNを選択する場合**：
- ハードウェアリソースが限られている
- リアルタイム性の要求が高くない
- 高い精度が必要

**Faster R-CNNを選択する場合**：
- 一般的なオブジェクト検出タスク
- 精度優先、ほぼリアルタイムで可
- 十分なハードウェアリソース（GPU推奨）
- エンドツーエンドトレーニングが必要

### 6.7 その後の発展

Faster R-CNNの後、二段階検出アルゴリズムはさらに発展：

- **Mask R-CNN** (2017): Faster R-CNNにインスタンスセグメンテーションブランチを追加
- **Cascade R-CNN** (2018): カスケード構造で段階的にIoU閾値を向上
- **FPN (Feature Pyramid Network)** (2017): マルチスケール特徴融合
- **各種改良版**: バックボーン最適化、注意メカニズムなど

### 6.8 単一段階アルゴリズムとの比較

| 比較次元 | 二段階 (Faster R-CNN) | 単一段階 (YOLO, SSD) |
|----------|----------------------|--------------------|
| 精度 | より高い | やや低い |
| 速度 | 遅い (5-10 fps) | 高速 (30-60+ fps) |
| 小オブジェクト検出 | より良い | やや劣る |
| トレーニングの難易度 | やや高い | 低い |
| リアルタイム性 | 達成が困難 | 達成が容易 |

## 付録：重要な用語

- **IoU (Intersection over Union)**: 交差和比、2つのボックスの重なり度を測るための指標
- **mAP (mean Average Precision)**: 平均精度の平均、オブジェクト検出の主要な評価指標
- **RoI (Region of Interest)**: 興味領域
- **Anchor**: 事前に設定された候補ボックス、検出ボックスの初期化に使用
- **Selective Search**: 従来の候補領域生成アルゴリズム
- **Backbone**: バックボーン、特徴抽出のためのCNN（VGG、ResNetなど）
- **Ground Truth**: 真のアノテーションデータ
- **Positive/Negative Sample**: 正サンプル/負サンプル
