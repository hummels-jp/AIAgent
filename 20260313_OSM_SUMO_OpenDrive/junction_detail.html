OSM / SUMO / OpenDRIVE 路口属性对比分析

🗺️ kawasaki.osm - 路口属性

⚠️ 重要说明: OSM格式中路口由多个道路相交的节点隐式表示,没有专门的路口元素。

隐式路口表示

OSM通过以下方式表示路口:

  • 多个 way 元素共享同一个 node 节点
  • 节点可能带有 junction 标签
  • 使用 relation 元素表示复杂路口

node 元素 - 路口节点

属性名 类型 说明
id 整数 节点的唯一标识符
lat 浮点数 纬度 (WGS84坐标)
lon 浮点数 经度 (WGS84坐标)

tag 元素 - 路口标签

标签名 (k) 值 (v) 说明
highway traffic_signals 交通信号灯路口
highway stop 停止标志路口
highway give_way 让行路口
junction roundabout 环岛
junction circular 环形路口
junction yes 路口标记
name 字符串 路口名称
ref 字符串 路口编号
turn:lanes 字符串 转向车道指示
turn:lanes:forward 字符串 前行车道转向
turn:lanes:backward 字符串 后行车道转向
restriction 字符串 转弯限制
except 字符串 限制例外

示例数据 片段 - 交通灯路口:

<!-- 路口节点 (交通灯) --> <node id="305945949" lat="35.5487398" lon="139.6592221"> <tag k="highway" v="traffic_signals"/> </node>

relation 元素 - 路口关系

对于复杂路口,使用 relation 元素表示:

标签名 (k) 值 (v) 说明
type restriction 转弯限制关系
type junction 路口关系
restriction no_left_turn 禁止左转
restriction no_right_turn 禁止右转
restriction no_u_turn 禁止掉头
restriction only_left_turn 只能左转
restriction only_right_turn 只能右转
restriction only_straight_on 只能直行

示例数据 片段 - 转弯限制:

<!-- 转弯限制关系 --> <relation id="123456"> <member type="way" ref="12345" role="from"/> <member type="node" ref="67890" role="via"/> <member type="way" ref="12346" role="to"/> <tag k="type" v="restriction"/> <tag k="restriction" v="no_left_turn"/> </relation>
💡 关键特点:
  • 通过道路相交隐式表示路口
  • 使用标签标记路口类型
  • 通过 relation 表示转弯限制
  • 不包含 路口几何形状
  • 不定义车道连接关系
  • 适用于地图显示和导航

🚗 kawasaki.net.xml - 路口属性

⚠️ 重要说明: SUMO格式使用 <junction> 元素明确定义路口,包含完整的连接和转向信息。

junction 元素 - 路口定义

属性名 类型 说明
id 字符串 路口唯一标识符
type 字符串 路口类型
x 浮点数 路口中心 X 坐标 (UTM)
y 浮点数 路口中心 Y 坐标 (UTM)
z 浮点数 路口中心 Z 坐标 (高度,可选)
incLanes 字符串 进入车道列表 (空格分隔)
intLanes 字符串 内部车道列表
shape 坐标序列 路口几何形状
路口类型 (type) 可选值:
priority - 优先级路口 (有让行规则)
traffic_light - 交通灯路口
right_before_left - 右侧先行
unregulated - 无规则路口
allway_stop - 全向停车
rail_crossing - 铁路平交
priority_stop - 停车优先

request 子元素 - 请求优先级

定义路口中各车道请求的优先级关系

属性名 类型 说明
index 整数 请求索引
response 字符串 响应关系 (0=禁止, 1=允许)
foes 字符串 冲突车道索引

connection 子元素 - 转向连接

属性名 类型 说明
from 字符串 进入边ID
to 字符串 离开 边ID
fromLane 整数 进入车道索引
toLane 整数 离开 车道索引
via 字符串 中间边ID (路口内部)
dir 字符串 转向方向 (s=直行, l=左转, r=右转, t=掉头)
state 字符串 连接状态 (c=连续, m=马蹄形)
type 字符串 连接类型
tl 字符串 关联的交通灯ID
linkIndex 整数 信号控制索引
keepClear 布尔 是否保持畅通
uncontrolled 布尔 是否无控制
length 浮点数 连接长度

示例数据 片段 - 交通灯路口:

<!-- 交通灯路口定义 --> <junction id="1389183173" type="traffic_light" x="745.23" y="567.89" z="0.00" incLanes="1389183173_0 1389183173_1 1389183167_0 1389183167_1" intLanes=":1389183173_0_0 :1389183173_1_0 :1389183173_2_0 :1389183173_3_0" shape="740.15,563.24 745.23,562.89 747.56,565.34 743.45,567.89"> <!-- 连接1: 从边0直行到边1 --> <connection from="1389183173" to="1389183167" fromLane="0" toLane="0" via=":1389183173_0" dir="s" state="c" tl="cluster_1389183173_1389183167" linkIndex="0" keepClear="true"/> <!-- 连接2: 从边0右转到边2 --> <connection from="1389183173" to="2860" fromLane="0" toLane="-1" via=":1389183173_1" dir="r" state="c" tl="cluster_1389183173_1389183167" linkIndex="1" keepClear="true"/> <!-- 连接3: 从边1左转到边3 --> <connection from="1389183167" to="1389183174" fromLane="0" toLane="-1" via=":1389183173_2" dir="l" state="c" tl="cluster_1389183173_1389183167" linkIndex="2" keepClear="true"/> <!-- 连接4: 从边2掉头 --> <connection from="2860" to="2860" fromLane="-1" toLane="-1" via=":1389183173_3" dir="t" state="c" tl="cluster_1389183173_1389183167" linkIndex="3" keepClear="true"/> <!-- 请求优先级定义 --> <request index="0" response="0000" foes="0000"/> <request index="1" response="1000" foes="0110"/> <request index="2" response="0100" foes="1001"/> <request index="3" response="0010" foes="0101"/> </junction>
💡 关键特点:
  • 明确的
    路口拓扑
    结构
  • 车道级别的连接定义
  • 完整的转向信息 (s/l/r/t)
  • 交通灯集成 (tl/linkIndex)
  • 冲突检测和优先级
  • 路口几何形状
  • 适用于精确的交通仿真

🛣️ kawasaki.xodr - 路口属性

⚠️ 重要说明: OpenDRIVE格式使用 <junction> 元素定义路口,包含道路连接的详细拓扑关系 。

junction 元素 - 路口定义

属性名 类型 说明
id 字符串 路口唯一标识符
name 字符串 路口名称
type 字符串 路口类型
s 浮点数 起始位置 (可选)
路口类型 (type) 可选值:
default - 标准路口
virtual - 虚拟路口 (道路连接点)
direct - 直接连接

connection 子元素 - 道路连接

属性名 类型 说明
id 字符串 连接ID
incomingRoad 字符串 进入道路ID
connectingRoad 字符串 连接道路ID (路口内部)
contactPoint 字符串 接触点 (start/end)

laneLink 子元素 - 车道连接

属性名 类型 说明
from 整数 进入车道ID
to 整数 离开 车道ID

controller 子元素 - 信号控制器

属性名 类型 说明
id 字符串 控制器ID
name 字符串 控制器名称
sequence 整数 控制顺序

junctionGroup 子元素 - 路口分组

属性名 类型 说明
id 字符串 路口组ID
name 字符串 路口组名称
type 字符串 分组类型 (roundabout等)

示例数据 片段:

<!-- 路口定义 --> <junction name="" id="1" type="default"> <!-- 连接1: 从道路2860到路口内部道路 --> <connection id="0" incomingRoad="2860" connectingRoad="1" contactPoint="end"> <!-- 车道连接: 第1车道到第1车道 --> <laneLink from="-1" to="-1"/> </connection> <!-- 连接2: 从道路2873到路口内部道路 --> <connection id="1" incomingRoad="2873" connectingRoad="1" contactPoint="start"> <!-- 车道连接: 第1车道到第1车道 --> <laneLink from="-1" to="-1"/> </connection> <!-- 连接3: 从道路2874到路口内部道路 --> <connection id="2" incomingRoad="2874" connectingRoad="1" contactPoint="end"> <!-- 车道连接: 第1车道到第1车道 --> <laneLink from="-1" to="-1"/> </connection> <!-- 连接4: 从道路2875到路口内部道路 --> <connection id="3" incomingRoad="2875" connectingRoad="1" contactPoint="end"> <!-- 车道连接: 第1车道到第1车道 --> <laneLink from="-1" to="-1"/> </connection> <!-- 信号控制器 --> <controller id="1" sequence="0"/> </junction>

junctionType 路口类型

类型 说明
default 标准路口
virtual 虚拟路口 (道路连接点,不包含 实际路口几何)
direct 直接连接 (道路直接相连)
💡 关键特点:
  • 清晰的
    路口拓扑
    结构
  • 车道级别的连接 (laneLink)
  • 支持路口分组
  • 集成信号控制器
  • 支持虚拟路口
  • 适用于高精度自动驾驶仿真
  • 内部道路显式定义

📊 三种格式路口属性对比总结

🗺️ OSM - 隐式表示

  • 道路相交隐式表示
  • 标签标记路口类型
  • relation表示转弯限制
  • 无几何形状
  • 适用于导航

🚗 SUMO - 完整拓扑

  • 明确路口定义
  • 车道级连接
  • 完整转向信息
  • 冲突检测
  • 交通灯集成

🛣️ OpenDRIVE - 高精度

  • 清晰拓扑结构
  • laneLink连接
  • 路口分组支持
  • 内部道路定义
  • 控制器集成
