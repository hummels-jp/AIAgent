OSM / SUMO / OpenDRIVE 行人设施属性对比分析

🗺️ kawasaki.osm - 行人设施属性

⚠️ 重要説明: OSMフォーマット中行人设施主要通过 highway 标签和专用元素表示。

way 元素 - 人行步道

标签名 (k) 值 (v) 说明
highway footway 人行道/步道
highway path 小径/非铺装路
highway pedestrian 步行区域/步行街
highway steps 楼梯
highway cycleway 自行车道 (可能允许行人)
highway track 轨道/小径

node 元素 - 人行横道和公交站

标签名 (k) 值 (v) 说明
highway crossing 人行横道标记
highway bus_stop 公交车站
highway stop 公交站点

补充标签

标签名 (k) 值 (v) 说明
surface 字符串 路面类型 (paved, asphalt, concrete, gravel等)
width 浮点数 宽度 (米)
lit yes/no 是否有照明
tactile_paving yes/no 是否有盲道
handrail yes/no/left/right/both 是否有扶手
stroller yes/no 是否适合婴儿车
wheelchair yes/no/limited 是否适合轮椅
incline up/down/yes/no 坡度方向
step_count 整数 楼梯台阶数
crossing unmarked/zebra/island 人行横道类型
crossing:island yes/no 是否有安全岛
crossing:signals yes/no 是否有行人信号灯
bench yes/no 是否有长椅
shelter yes/no 是否有遮雨棚
bin yes/no 是否有垃圾桶

示例数据 片段 - 人行横道:

<!-- 人行横道节点 --> <node id="306218670" lat="35.5501234" lon="139.6615678"> <tag k="highway" v="crossing"/> <tag k="crossing" v="zebra"/> <tag k="crossing:signals" v="yes"/> <tag k="tactile_paving" v="yes"/> </node>

示例数据 片段 - 公交站:

<!-- 公交车站节点 --> <node id="12345678" lat="35.5512345" lon="139.6626789"> <tag k="highway" v="bus_stop"/> <tag k="name" v="川崎駅前"/> <tag k="public_transport" v="stop_position"/> <tag k="bench" v="yes"/> <tag k="shelter" v="yes"/> <tag k="wheelchair" v="yes"/> </node>

示例数据 片段 - 人行道:

<!-- 人行道 --> <way id="98765432"> <nd ref="11111111"/> <nd ref="22222222"/> <nd ref="33333333"/> <tag k="highway" v="footway"/> <tag k="surface" v="paved"/> <tag k="width" v="2.0"/> <tag k="lit" v="yes"/> <tag k="tactile_paving" v="yes"/> <tag k="wheelchair" v="yes"/> </way>
💡 关键特徴:
  • 使用 highway 标签区分设施类型
  • 丰富的辅助标签补充信息
  • 支持なし障碍设施标记
  • 适用于导航和地图显示
  • 不包含 精确几何形状
  • 灵活性高,依赖数据质量

🚗 kawasaki.net.xml - 行人设施属性

⚠️ 重要説明: SUMOフォーマット通过 edge 类型 、allow 属性和 walkingareas 表示行人设施。

type 元素 - 人行道类型

属性名 类型 说明
id 字符串 类型 ID (如: highway.footway)
priority 整数 优先级 (通常为1)
numLanes 整数 车道数 (通常为1)
speed 浮点数 行人速度 (m/s, 约1.0-1.5)
allow 字符串 允许的车辆类型 (pedestrian)
disallow 字符串 禁止的车辆类型
width 浮点数 宽度 (米)
oneway 0/1 是否单向 (通常为0)

示例数据 片段 - footway 类型 :

<!-- 人行道类型 定义 --> <type id="highway.footway" priority="1" numLanes="1" speed="1.39" allow="pedestrian" oneway="1" width="2.00"/>

edge 元素 - 人行道边

属性名 类型 说明
id 字符串 边ID
from 字符串 起始节点ID
to 字符串 结束节点ID
type 字符串 边类型 (highway.footway)
shape 坐标序列 几何形状 (UTM坐标)

lane 元素 - 人行道车道

属性名 类型 说明
id 字符串 车道ID
index 整数 车道索引
allow 字符串 允许通行: pedestrian
speed 浮点数 行人速度 (m/s)
length 浮点数 长度 (米)
width 浮点数 宽度 (米)
shape 坐标序列 车道几何形状

walkingarea 元素 - 行人区域

属性名 类型 说明
id 字符串 区域ID
shape 坐标序列 区域几何形状 (多边形)

示例数据 片段 - 人行道边:

<!-- 人行道边 --> <edge id=":13315440577_0" function="internal"> <!-- 允许行人、配送车辆、自行车通行 --> <lane id=":13315440577_0_0" index="0" allow="pedestrian delivery bicycle" speed="1.39" length="5.84" shape="395.44,711.00 401.27,710.68"/> </edge>

车辆类型

类型 说明
pedestrian 行人
bicycle 自行车
delivery 配送车辆
emergency 紧急车辆
passenger 客运车辆
bus 公交车
tram 有轨电车
💡 关键特徴:
  • 明确的类型 定义
  • allow/disallow 精细控制通行权
  • 支持混合通行
  • 几何形状精确
  • 支持行人区域
  • 适用于交通仿真
  • 速度基于行人行为模型

🛣️ kawasaki.xodr - 行人设施属性

⚠️ 重要説明: OpenDRIVEフォーマット通过车道类型 、road类型 和objects元素表示行人设施。

lane 元素 - 车道类型

类型 (type) 说明
sidewalk 人行道
walking 步行区域
crossing 人行横道
border 边界 (可能含人行道)
shoulder 路肩 (可能含人行道)
biking 自行车道
restricted 限制区域

road 元素 - 道路类型

类型 说明
walking 步行街
town 城市道路 (可能含人行道)
rural 乡村道路
lowSpeed 低速道路

objects 元素 - 行人相关设施

属性名 类型 说明
id 字符串 对象唯一标识符
s 浮点数 沿道路的纵向位置 (s坐标)
t 浮点数 横向偏移 (t坐标)
zOffset 浮点数 垂直偏移 (高度)
validLength 浮点数 有效长度
orientation 字符串 朝向 (+/-)
type 字符串 对象类型
name 字符串 对象名称
dynamic 字符串 是否动态对象
常见对象类型 :
barrier - 栏杆
busStop - 公交站
crossing - 人行横道
fence - 围栏
gate - 大门
pole - 电线杆
railing - 栏杆
tree - 树木
wall - 墙壁
obstacles - 障碍物

object 子元素 - busStop (公交站)

属性名 类型 说明
s 浮点数 沿道路的纵向位置
t 浮点数 横向偏移
id 字符串 公交站ID
type 字符串 类型 (busStop)
name 字符串 公交站名称
lane 整数 关联车道ID

示例数据 片段 - 人行道:

<!-- 道路包含人行道 --> <road name="" length="176.38676806" id="2873"> <lanes> <laneSection s="0"> <right> <!-- 行驶车道 --> <lane id="-1" type="driving" level="true"> <link/> <width sOffset="0" a="3.20" b="0" c="0" d="0"/> </lane> <!-- 人行道 --> <lane id="-2" type="sidewalk" level="true"> <link/> <width sOffset="0" a="2.00" b="0" c="0" d="0"/> <roadMark sOffset="0" type="solid" color="standard"/> </lane> </right> </laneSection> </lanes> <!-- 行人相关对象 --> <objects> <!-- 公交站 --> <object s="50.00" t="-4.50" zOffset="0.15" id="busstop_2873_1" type="busStop" name="川崎駅前" dynamic="yes"> <repeat s="50.00" length="10.00" distance="0"/> </object> <-- 栏杆 --> <object s="0.00" t="-5.50" zOffset="0.80" validLength="176.39" id="railing_2873_1" type="railing"> <repeat s="0.00" length="176.39" distance="0"/> </object> </objects> </road>

示例数据 片段 - 步行街:

<!-- 步行街道路 --> <road name="商店街" length="100.00" id="3000" type="walking"> <link> <predecessor elementType="junction" elementId="100"/> <successor elementType="junction" elementId="101"/> </link> <planView> <!-- 几何定义... --> </planView> <lanes> <laneSection s="0"> <center> <lane id="0" type="walking"> <link/> <width sOffset="0" a="5.00"b="0"c="0"d="0"/> </lane> </center> </laneSection> </lanes> </road>
💡 关键特徴:
  • 多种车道类型 表示行人设施
  • 支持专门的walking道路类型
  • 丰富的objects定义设施
  • 精确的几何和位置
  • 支持动态对象
  • 适用于高精度自动驾驶仿真
  • 支持行人-车辆交互建模

📊 三种フォーマット行人设施属性对比总结

🗺️ OSM - 标签系统

  • highway标签区分
  • 丰富的辅助标签
  • なし障碍设施标记
  • 适用于导航
  • 依赖数据质量

🚗 SUMO - 类型 控制

  • 明确类型 定义
  • allow/disallow控制
  • 混合通行支持
  • 行人区域
  • 交通仿真

🛣️ OpenDRIVE - 类型 +对象

  • 多种车道类型
  • walking道路
  • 丰富objects
  • 精确几何
  • 自动驾驶仿真
