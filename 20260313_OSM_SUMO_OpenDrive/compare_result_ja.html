🗺️ 地図要素分類比較

🛣️ 1. 道路中心線

kawasaki.osm

使用 way 元素表示道路,通过 nd 引用节点序列来定义道路几何形状。
主要な属性:
  • highwayタグ: 道路タイプ (primary, residential, tertiary等)
  • nameタグ: 道路名（多言語対応）
  • onewayタグ: 一方通行標識
  • maxspeedタグ: 速度制限情報
  • lanesタグ: 車線数
詳細を表示 →

kawasaki.net.xml

使用 edge 元素表示道路边,包含几何形状和连接信息。
主要な属性:
  • id: エッジ識別子
  • from/to: 始点/終点ノードID
  • shape: 幾何形状座標シーケンス
  • length: 道路長（メートル）
  • priority: 道路優先度
  • numLanes: 車線数
詳細を表示 →

kawasaki.xodr

使用 road 元素表示道路,包含高精度的几何和拓扑信息。
主要な属性:
  • length: 道路の正確な長さ
  • junction: 交差点ID
  • planView/geometry: 道路幾何（直線、曲線、スパイラル対応）
  • link: 道路接続関係 (predecessor/successor)
  • elevationProfile: 標高プロファイル

🚗 2. 車線

kawasaki.osm

通过 lanes 标签简单表示车道数量,没有详细的车道几何信息。
特徴:
  • 車線総数のみを記録し、具体的な車線を区別しない
  • なし法表示車線幅变化
  • 車線接続情報を含まない
  • なし法表示特殊车道(如公交专用道)
詳細を表示 →

kawasaki.net.xml

使用 lane 元素详细描述每条车道,包含车道级别信息。
主要な属性:
  • index: 車線インデックス番号
  • speed: 車線速度制限（m/s）
  • length: 车道长度
  • width: 車線幅
  • allow/disallow: 許可/禁止される車両タイプ
  • shape: 車線幾何形状
詳細を表示 →

kawasaki.xodr

车道信息嵌套在 laneSection 中,支持最复杂的车道配置。
主要な属性:
  • left/center/right: 左/中/右車線グループ
  • width: 道路に沿った幅変化関数
  • roadMark: 標線タイプ（実線、破線、色）
  • speed: 速度制限曲线
  • link: 車線リンク関係（laneLink）

🔀 4. 交差点

kawasaki.osm

通过多个 way 相交的节点隐式表示路口,没有专门的路口元素。
特徴:
  • 道路の収束で交差点を表現
  • 可能使用 junction 标签标记路口类型
  • 曲がり接続情報を含まない
  • 曲がり制限を定義しない

主要な属性:

  • incLanes: 進入車線リスト
  • intLanes: 内部車線リスト
  • shape: 交差点形状

  • connection: 道路连接(incomingRoad, connectingRoad)
  • laneLink: 车道级链接
  • controller: 信号控制器

🛑 6. 道路標線

kawasaki.osm

標線情報は限られており、主に道路タイプと車線数から推測される。
特徴:
  • 通常は標線を直接表現しない
  • 特定のタグで特殊な標線をマークする場合がある
  • 道路タイプと車線数に依存
  • なし标线颜色和样式信息

標線情報は少なく、主に車線設定で表現される。
特徴:

🔍 OpenDRIVEファイル詳細比較: SUMO vs DGM

⚠️ 説明: dgm_kawasaki.xodr 文件覆盖了更广阔的地理范围,包含更多的道路和路口元素,因此ファイルサイズ和行数差异较大。同时该文件使用了更新的OpenDRIVE 1.5標準,提供了更丰富的扩展属性。

✅ 优点

  • ファイルがよりコンパクト、転送と保存に便利
  • 包含完整的速度制限信息
  • 使用UTM局部座標系统,易于处理
  • 車線構造が比較的単純、解析が容易
  • SUMOシミュレーションでの直接使用に適している
  • 左側通行ルール（lefthand=true）

❌ 缺点

  • OpenDRIVEバージョンが古い（1.4）
  • 交差点数较少
  • 缺少surface要素
  • 缺少userData拡張信息
  • 数値精度が比較的低い
  • なし车道高度信息(height属性)

✅ 优点

  • 最新のOpenDRIVE 1.5仕様を使用
  • より多くの交差点定義を含む（285 vs 217）
  • 詳細な車線マーキング（height、laneChange）
  • 每个道路包含surface要素
  • userData/viStyleDef拡張データを含む
  • 極めて高い数値精度（科学表記法、16桁）
  • すべての道路がrule属性を含む

❌ 缺点

  • ファイルサイズが大きい（地理的範囲が広い）
  • 缺少速度制限信息
  • なしgeoReference座標系统定义
  • 直接地理座標を使用、変換が必要
  • 車線構造がより複雑、解析が困難
  • SUMOでの直接使用には不向きの場合がある
