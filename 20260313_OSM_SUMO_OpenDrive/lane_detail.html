OSM / SUMO / OpenDRIVE 车道属性对比分析

🗺️ kawasaki.osm - 车道属性

⚠️ 重要说明: OSM格式中不直接定义车道级别的几何信息,仅通过标签简单表示
车道数量
和基本属性。

way 元素 - 道路基础属性

属性名 类型 说明
id 整数 way元素的唯一标识符
visible 布尔 是否可见 (true/false)
version 整数 版本号,用于追踪编辑历史
changeset 整数 变更集ID
timestamp ISO 8601时间 最后更新时间
user 字符串 最后编辑用户名
uid 整数 编辑用户ID

示例数据 片段:

<!-- 道路基础属性 --> <way id="28918313" visible="true" version="6" changeset="141192260" timestamp="2023-09-13T06:38:38Z" user="tg4567" uid="12500589"> <nd ref="13318651276"/> <nd ref="13318651278"/> <nd ref="306218670"/> <!-- 更多节点引用... --> </way>

tag 元素 - 车道相关标签

标签名 (k) 值 (v) 说明
highway primary, residential, tertiary等 道路类型 ,影响
车道数量
和宽度默认值
lanes 整数 车道总数 (如: "2")
lanes:forward 整数 前行方向车道数
lanes:backward 整数 后行方向车道数
lanes:both_ways 整数 双向车道数
width:lanes 字符串 车道宽度,多个宽度用 | 分隔
turn:lanes 字符串 每条车道的转向指示 (如: "left|through|right")
turn:lanes:forward 字符串 前行车道转向指示
lane_markings yes/no 是否有车道标线
hgv yes/no/designated 是否允许重型卡车通行
bus yes/no/designated 是否允许公交通行
psv yes/no/designated 是否允许公共交通车辆

示例数据 片段:

<!-- 道路车道标签 --> <tag k="highway" v="primary"/> <tag k="lanes" v="2"/> <tag k="maxspeed" v="40"/> <tag k="oneway" v="no"/> <tag k="cycleway" v="lane"/> <tag k="surface" v="paved"/>
💡 关键特点:
  • 仅记录车道总数,不区分具体车道
    的几何形状
  • 无法表示车道宽度沿道路的变化
  • 不包含 车道连接关系 和转向限制
  • 使用简单标签系统,灵活性高但精度有限
  • 适用于地图显示和基础导航,不适用于精确仿真

🚗 kawasaki.net.xml - 车道属性

⚠️ 重要说明: SUMO网络格式使用 <lane> 元素详细定义每条车道的属性,包括速度、宽度、长度和允许的车辆类型 。

edge 元素 - 道路边属性

属性名 类型 说明
id 字符串 边的唯一标识符
from 字符串 起始节点ID
to 字符串 结束节点ID
priority 整数 道路优先级 (1-12)
type 字符串 道路类型 (如: highway.primary)
function 字符串 边的功能 (normal, internal, connector等)
shape 坐标序列 边的几何形状 (UTM坐标)
spreadType 字符串 车道分布方式 (center, right等)

lane 子元素 - 车道详细属性

属性名 类型 说明
id 字符串 车道唯一ID (格式: edgeID_laneIndex)
index 整数 车道索引号 (从右向左,从0开始)
speed 浮点数 车道限速 (m/s)
length 浮点数 车道长度 (米)
width 浮点数 车道宽度 (米),可选
shape 坐标序列 车道中心线几何形状 (UTM坐标)
allow 字符串 允许通行的车辆类型 (空格分隔)
disallow 字符串 禁止通行的车辆类型 (空格分隔)
accels 浮点数 加速度 ,可选

示例数据 片段 - type 定义:

<!-- 道路类型 定义 --> <type id="highway.footway" priority="1" numLanes="1" speed="2.78" allow="pedestrian" oneway="1" width="2.00"/> <type id="highway.primary" priority="12" numLanes="2" speed="27.78" disallow="tram rail_urban rail rail_electric rail_fast ship container cable_car subway aircraft wheelchair scooter drone" oneway="0"/> <type id="highway.residential" priority="3" numLanes="1" speed="13.89" disallow="tram rail_urban rail rail_electric rail_fast ship container cable_car subway aircraft wheelchair scooter drone" oneway="0"/>

示例数据 片段 - edge 和 lane 定义:

<!-- 路口内部边 --> <edge id=":13315440559_0" function="internal"> <lane id=":13315440559_0_0" index="0" disallow="tram rail_urban rail rail_electric rail_fast ship container cable_car subway aircraft wheelchair scooter drone" speed="13.89" length="2.12" shape="743.01,873.76 740.94,874.22"/> </edge> <!-- 服务道路 --> <edge id=":13315440574_0" function="internal"> <lane id=":13315440574_0_0" index="0" disallow="tram rail_urban rail rail_electric rail_fast ship container cable_car subway aircraft wheelchair scooter drone" speed="13.89" length="5.84" shape="391.27,746.27 385.44,746.58"/> </edge> <!-- 允许行人、配送车、自行车的车道 --> <edge id=":13315440577_0" function="internal"> <lane id=":13315440577_0_0" index="0" allow="pedestrian delivery bicycle" speed="5.56" length="5.84" shape="395.44,711.00 401.27,710.68"/> </edge> <!-- 次级道路,速度22.22 m/s (80 km/h) --> <edge id=":13315440583_0" function="internal"> <lane id=":13315440583_0_0" index="0" disallow="tram rail_urban rail rail_electric rail_fast ship container cable_car subway aircraft wheelchair scooter drone" speed="22.22" length="6.36" shape="424.08,634.48 417.72,634.75"/> </edge>
💡 关键特点:
  • 支持车道级别的几何定义和速度限制
  • 使用 allow/disallow 属性精细控制车辆类型
  • 车道索引系统: 从道路右侧向左侧递增
  • 支持 UTM 投影坐标系 ,适用于仿真
  • 可定义路口内部的连接车道
  • 速度单位为 m/s,需要换算 (如: 22.22 m/s ≈ 80 km/h)

🛣️ kawasaki.xodr - 车道属性

⚠️ 重要说明: OpenDRIVE 格式支持最复杂的
车道配置
,包括沿道路变化的宽度、标线类型 、速度曲线等。

road 元素 - 道路基础属性

属性名 类型 说明
id 字符串 道路唯一标识符
name 字符串 道路名称
length 浮点数 道路精确长度 (米)
junction 整数 所属路口ID (-1表示非路口)

lanes/laneSection 元素 - 车道段 定义

元素/属性 类型 说明
laneSection/s 浮点数 车道段 起始位置 (沿道路的s坐标)
center 元素 中心车道 (通常为参考线)
right 元素 右侧车道组 (负索引: -1, -2, ...)
left 元素 左侧车道组 (正索引: 1, 2, ...)

lane 元素 - 车道详细属性

属性名 类型 说明
id 整数 车道ID (正数表示左侧,负数表示右侧,0为中心)
type 字符串 车道类型
level 布尔 车道是否在同一平面
车道类型 (type) 可选值:
none 无类型
driving 行驶车道
stop 停止车道
shoulder 路肩
biking 自行车道
sidewalk 人行道
border 边界
restricted 限制车道
parking 停车位
median 中央隔离带
walking 步行区域

width 子元素 - 车道宽度

属性名 类型 说明
sOffset 浮点数 起始位置偏移 (沿道路的s坐标)
a 浮点数 多项式常数项 (基准宽度)
b 浮点数 多项式一次项系数
c 浮点数 多项式二次项系数
d 浮点数 多项式三次项系数
宽度公式: width(s) = a + b·ds + c·ds² + d·ds³, 其中 ds = s - sOffset

roadMark 子元素 - 道路标线

属性名 类型 说明
sOffset 浮点数 起始位置偏移
type 字符串 标线类型
weight 字符串 线条粗细 (standard, bold)
color 字符串 标线颜色
width 浮点数 标线宽度 (米)
laneChange 字符串 是否允许变道 (increase, decrease, both, none)
标线类型 (type) 可选值:
none 无标线
solid 实线
broken 虚线
solid solid 双实线
solid broken 实虚线
broken solid 虚实线
broken broken 双虚线
botts dots 道路凸起标记
grass 草地
颜色 (color) 可选值:
standard 标准 (默认)
white 白色
yellow 黄色
red 红色
blue 蓝色

speed 子元素 - 速度限制

属性名 类型 说明
sOffset 浮点数 起始位置偏移
max 浮点数 最大速度 (m/s)

link 子元素 - 车道链接

子元素 说明
predecessor 连接的前驱车道
successor 连接的后继车道

示例数据 片段:

<!-- 道路定义 --> <road name="" length="107.64497470" id="2860" junction="-1"> <link> <predecessor elementType="junction" elementId="1"/> <successor elementType="junction" elementId="2"/> </link> <type s="0" type="town"/> <planView> <!-- 几何定义... --> </planView> <!-- 车道段 定义 --> <lanes> <laneSection s="0"> <!-- 中心车道 --> <center> <lane id="0" type="none" level="true"> <link/> <roadMark sOffset="0" type="solid" weight="standard" color="standard" width="0.13"/> </lane> </center> <!-- 右侧行驶车道 --> <right> <lane id="-1" type="driving" level="true"> <link/> <!-- 宽度: 3.20米,不随位置变化 (a=3.20, b=c=d=0) --> <width sOffset="0" a="3.20" b="0" c="0" d="0"/> <!-- 标线: 实线,标准粗细,标准颜色 --> <roadMark sOffset="0" type="solid" weight="standard" color="standard" width="0.13"/> <!-- 速度限制: 13.89 m/s (50 km/h) --> <speed sOffset="0" max="13.89"/> </lane> </right> </laneSection> </lanes> </road>

示例数据 片段 - 限制车道:

<!-- 服务道路,限制车道 --> <road name="" length="0.20000000" id="2881" junction="-1"> <link> <predecessor elementType="junction" elementId="30"/> <successor elementType="junction" elementId="31"/> </link> <type s="0" type="town"/> <lanes> <laneSection s="0"> <center> <lane id="0" type="none" level="true"> <link/> <roadMark sOffset="0" type="solid" weight="standard" color="standard" width="0.13"/> </lane> </center> <right> <!-- 限制车道: 宽度3.20米,实线标线,速度5.56 m/s (20 km/h) --> <lane id="-1" type="restricted" level="true"> <link/> <width sOffset="0" a="3.20" b="0" c="0" d="0"/> <roadMark sOffset="0" type="solid" weight="standard" color="standard" width="0.13"/> <speed sOffset="0" max="5.56"/> </lane> </right> </laneSection> </lanes> </road>
💡 关键特点:
  • 支持沿道路变化的宽度函数 (三次多项式)
  • 详细的道路标线定义 (类型 、颜色、粗细)
  • 速度限制可以沿道路变化
  • 支持多种车道类型 (driving, sidewalk, biking等)
  • 明确的车道连接关系 (predecessor/successor)
  • 车道ID系统: 正数为左侧,负数为右侧,0为中心
  • 支持 laneChange 属性控制变道权限
  • 适用于高精度自动驾驶仿真

📊 三种格式车道属性对比总结

🗺️ OSM - 简单标签

  • 仅记录车道总数
  • 通过标签表示基本属性
  • 无车道几何信息
  • 无法表示宽度变化
  • 适用于地图显示

🚗 SUMO - 仿真 级

  • 车道级别属性
  • 几何形状和速度
  • 车辆类型 控制
  • 固定车道宽度
  • 适用于交通仿真

🛣️ OpenDRIVE - 高精度

  • 宽度变化函数
  • 详细标线定义
  • 速度曲线
  • 多种车道类型
  • 适用于自动驾驶
