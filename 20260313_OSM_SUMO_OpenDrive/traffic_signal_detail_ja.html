OSM / SUMO / OpenDRIVE 交通灯属性对比分析

🗺️ kawasaki.osm - 交通灯属性

⚠️ 重要説明: OSMフォーマット中交通灯仅作为位置标记,不包含 相位控制逻辑。

node 元素 - 交通灯标记

属性名 类型 说明
id 整数 节点的唯一标识符
lat 浮点数 纬度 (WGS84坐标)
lon 浮点数 经度 (WGS84坐标)
visible 布尔 是否可见 (true/false)
version 整数 版本号
timestamp ISO 8601时间 最后更新时间
user 字符串 最后编辑用户名

tag 元素 - 交通灯标签

标签名 (k) 值 (v) 说明
highway traffic_signals 标识为交通灯
name 字符串 交通灯名称
traffic_signals:direction 字符串 信号方向
traffic_signals:lanes 整数 控制的车道数
traffic_signals:count 整数 信号灯数量
crossing traffic_signals 人行横道信号灯
crossing:island yes/no 是否有安全岛
crossing:signals yes/no 是否有行人信号灯

示例数据 片段:

<!-- 交通灯节点 --> <node id="305945949" lat="35.5487398" lon="139.6592221" visible="true" version="2" changeset="141192260" timestamp="2023-09-13T06:38:38Z" user="tg4567" uid="12500589"> <tag k="highway" v="traffic_signals"/> </node>
💡 关键特徴:
  • 仅标记交通灯位置,なし相位控制信息
  • 不包含 红/黄/绿灯时长
  • なし法表示信号控制逻辑
  • 适用于地图显示,不适用于仿真
  • 可使用附加标签补充信息

🚗 kawasaki.net.xml - 交通灯属性

⚠️ 重要説明: SUMOフォーマット使用 <tlLogic> 元素定义完整的交通灯控制逻辑,包含相位定义。

tlLogic 元素 - 交通灯逻辑

属性名 类型 说明
id 字符串 交通灯唯一标识符
type 字符串 交通灯类型
programID 字符串 信号程序ID
offset 浮点数 时间偏移 (秒)
交通灯类型 (type) 可选值:
static - 静态信号
actuated - 感应控制
delay_based - 基于延迟

phase 子元素 - 相位定义

属性名 类型 说明
duration 浮点数 相位持续时间 (秒)
state 字符串 各车道信号状态 (r=红, y=黄, g=绿, s=黄红, o=关闭)
minDur 浮点数 最小持续时间 (可选)
maxDur 浮点数 最大持续时间 (可选)
信号状态字符:
r - 红灯 (禁止通行)
y - 黄灯 (即将变红)
g - 绿灯 (允许通行)
s - 黄红灯 (即将变绿)
o - 关闭 (なし信号)

示例数据 片段:

<!-- 交通灯逻辑定义 --> <tlLogic id="cluster_13315440559_1389183167" type="static" programID="0" offset="0"> <!-- 相位1: 所有方向红灯 --> <phase duration="2" state="rrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrr"/> <!-- 相位2: 主要方向绿灯,次要方向红灯 --> <phase duration="31" state="GGgrrrrrrrrrrrrrrGGggrrrrrrrr"/> <!-- 相位3: 黄灯 --> <phase duration="4" state="yyrrrrrrrrrrrrrrrryyyyrrrrrrr"/> <!-- 相位4: 次要方向绿灯,主要方向红灯 --> <phase duration="31" state="rrrrrrrrrrrGGgrrrrrrrrrrGGgg"/> <!-- 相位5: 黄灯 --> <phase duration="4" state="rrrrrrrrrrrryrrrrrrrrrrryyy"/> </tlLogic>

connection 元素 - 信号连接

在 junction 元素中,connection 元素指定每个转向对应的信号控制索引:

属性名 类型 说明
tl 字符串 关联的交通灯ID
linkIndex 整数 连接索引,对应 phase.state 中的位置
💡 关键特徴:
  • 完整的相位控制逻辑
  • 支持多种控制类型 (static/actuated/delay_based)
  • 精确的时间控制 (duration/offset)
  • 每个转向独立的信号状态
  • 支持多个信号程序 (programID)
  • 适用于精确的交通仿真

🛣️ kawasaki.xodr - 交通灯属性

⚠️ 重要説明: OpenDRIVEフォーマット使用 <signal> 元素定义交通灯,包含位置和类型 信息。

signal 元素 - 交通信号定义

属性名 类型 说明
id 字符串 信号唯一标识符
s 浮点数 沿道路的纵向位置 (s坐标,米)
t 浮点数 横向偏移 (t坐标,米)
dynamic 字符串 是否动态信号 (yes/no)
orientation 字符串 信号朝向 (+/-)
country 字符串 国家代码
type 字符串 信号类型
subtype 字符串 信号子类型
value 浮点数 信号值 (如限速)
unit 字符串 单位
height 浮点数 信号高度 (米)
hOffset 浮点数 水平偏移角度 (度)
vOffset 浮点数 垂直偏移角度 (度)
pitch 浮点数 俯仰角 (度)
roll 浮点数 翻滚角 (度)

常见信号类型

类型 说明
1000001 红灯
1000002 黄灯
1000003 绿灯
1000004 停止标志
1000005 让行标志
1000006 限速标志
1000007 禁止进入
1000008 单行道标志
1000009 禁止停车
1000010 禁止超车
1000011 人行横道
1000012 人行横道信号灯

示例数据 片段:

<!-- 交通信号定义 --> <signals> <!-- 交通灯 (红灯) --> <signal id="1389183173" s="45.67" t="-3.20" dynamic="yes" orientation="-" country="JPN" type="1000001" subtype="0" height="4.50" hOffset="0.00" vOffset="0.00" pitch="0.00" roll="0.00"> <validity fromLane="-1" toLane="-1"/> </signal> <!-- 人行横道信号灯 --> <signal id="1389183174" s="23.45" t="1.50" dynamic="yes" orientation="-" country="JPN" type="1000012" height="3.00"> <validity fromLane="-1" toLane="-1"/> </signal> </signals>

validity 子元素 - 适用范围

属性名 类型 说明
fromLane 整数 起始车道ID
toLane 整数 结束车道ID
fromRoad 字符串 起始道路ID
toRoad 字符串 结束道路ID

controller 元素 - 控制器

属性名 类型 说明
id 字符串 控制器ID
name 字符串 控制器名称
sequence 整数 控制顺序
💡 关键特徴:
  • 精确的3D位置 (s, t, height, 角度)
  • 丰富的信号类型 系统
  • 支持动态信号 (dynamic=yes)
  • 可指定信号适用范围 (validity)
  • 支持多个控制器
  • 适用于高精度自动驾驶仿真
  • 相位控制需要通过控制器外部定义

📊 三种フォーマット交通灯属性对比总结

🗺️ OSM - 位置标记

  • 仅标记位置
  • なし相位信息
  • なし控制逻辑
  • 适用于地图显示
  • 简单直观

🚗 SUMO - 完整控制

  • 完整相位逻辑
  • 精确时间控制
  • 多种控制类型
  • 适用于交通仿真
  • 状态机驱动

🛣️ OpenDRIVE - 位置+类型

  • 精确3D位置
  • 丰富信号类型
  • 动态信号支持
  • 适用于自动驾驶
  • 几何化定义
