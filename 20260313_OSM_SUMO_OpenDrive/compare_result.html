🗺️ 地图要素分类比较

🛣️ 1. 道路中心线

kawasaki.osm

使用 way 元素表示道路,通过 nd 引用节点序列来定义道路几何形状。
关键属性:
  • highway 标签: 道路类型 (primary, residential, tertiary等)
  • name 标签: 道路名称(多语言支持)
  • oneway 标签: 单向通行标识
  • maxspeed 标签: 限速信息
  • lanes 标签: 车道数量
查看详细属性 →

kawasaki.net.xml

使用 edge 元素表示道路边,包含几何形状和连接信息。
关键属性:
  • id: 边的标识符
  • from/to: 起止节点ID
  • shape: 几何形状坐标序列
  • length: 道路长度(米)
  • priority: 道路优先级
  • numLanes: 车道数量
查看详细属性 →

kawasaki.xodr

使用 road 元素表示道路,包含高精度的几何和拓扑信息。
关键属性:
  • length: 道路精确长度
  • junction: 所属路口ID
  • planView/geometry: 道路几何(支持直线、曲线、螺旋线)
  • link: 道路连接关系 (predecessor/successor)
  • elevationProfile: 高程剖面
查看详细属性 →

🚗 2. 车道

kawasaki.osm

通过 lanes 标签简单表示车道数量,没有详细的车道几何信息。
特点:
  • 仅记录车道总数,不区分具体车道
  • 无法表示车道宽度变化
  • 不包含车道连接信息
  • 无法表示特殊车道(如公交专用道)
查看详细属性 →

kawasaki.net.xml

使用 lane 元素详细描述每条车道,包含车道级别信息。
关键属性:
  • index: 车道索引号
  • speed: 车道限速(m/s)
  • length: 车道长度
  • width: 车道宽度
  • allow/disallow: 允许/禁止的车辆类型
  • shape: 车道几何形状
查看详细属性 →

kawasaki.xodr

车道信息嵌套在 laneSection 中,支持最复杂的车道配置。
关键属性:
  • left/center/right: 左/中/右车道分组
  • width: 沿道路的宽度变化函数
  • roadMark: 标线类型 (实线、虚线、颜色)
  • speed: 速度限制曲线
  • link: 车道链接关系 (laneLink)
查看详细属性 →

🚦 3. 交通灯

kawasaki.osm

使用 node 元素配合 highway=traffic_signals 标签标记交通灯位置。
特点:
  • 仅标记位置,无相位信息
  • 支持名称(name标签)
  • 不包含信号控制逻辑
  • 示例: <tag k="highway" v="traffic_signals"/>
查看详细属性 →

kawasaki.net.xml

使用 tlLogic 元素定义交通灯逻辑,包含完整的相位信息。
关键属性:
  • id: 交通灯ID
  • programID: 信号程序ID
  • phase: 相位定义(红/黄/绿状态)
  • duration: 相位持续时间
  • offset: 时间偏移
查看详细属性 →

kawasaki.xodr

使用 signals 元素定义信号,通常位于 junction 中。
关键属性:
  • id: 信号ID
  • s: 沿道路的纵向位置
  • t: 横向偏移
  • dynamic: 是否动态信号
  • country: 国家代码
  • type: 信号类型
查看详细属性 →

🔀 4. 路口

kawasaki.osm

通过多个 way 相交的节点隐式表示路口,没有专门的路口元素。
特点:
  • 通过道路汇聚表示路口
  • 可能使用 junction 标签标记路口类型
  • 不包含转弯连接信息
  • 不定义转弯限制
查看详细属性 →

kawasaki.net.xml

使用 junction 元素明确定义路口,包含完整的连接和转向信息。
关键属性:
  • id: 路口ID
  • type: 路口类型 (priority, traffic_light等)
  • incLanes: 进入车道列表
  • intLanes: 内部车道列表
  • shape: 路口形状
  • logic: 逻辑连接(connection元素)
查看详细属性 →

kawasaki.xodr

使用 junction 元素定义路口,包含连接道路的详细拓扑关系。
关键属性:
  • id: 路口ID
  • name: 路口名称
  • connection: 道路连接(incomingRoad, connectingRoad)
  • laneLink: 车道级链接
  • controller: 信号控制器
  • junctionGroup: 路口分组
查看详细属性 →

🚸 5. 行人设施

kawasaki.osm

使用 highway=footway 等标签表示人行道、步道等。
特点:
  • footway: 人行道
  • path: 小径
  • crossing: 人行横道
  • sidewalk: 人行道(作为道路属性)
  • 可能包含表面类型、宽度等信息
查看详细属性 →

kawasaki.net.xml

支持 pedestrian 元素类型,定义行人专用区域。
关键属性:
  • edge类型: highway.footway
  • allow: 允许pedestrian通行
  • width: 人行道宽度
  • speed: 行人速度
  • walkingareas: 行人区域(可选)
查看详细属性 →

kawasaki.xodr

通过 lane type="sidewalk" 或 type="walking" 表示行人设施。
关键属性:
  • lane type="sidewalk": 人行道
  • lane type="crossing": 人行横道
  • road type="walking": 步行街
  • objects: 行人相关设施(长椅、路灯等)
  • 支持障碍物表示
查看详细属性 →

🛑 6. 道路标线

kawasaki.osm

标线信息有限,主要通过道路类型和车道数推断。
特点:
  • 通常不直接表示标线
  • 可能使用特定标签标记特殊标线
  • 依赖道路类型和车道数
  • 无标线颜色和样式信息
查看详细属性 →

kawasaki.net.xml

标线信息较少,主要通过车道配置表示。
特点:
  • 不直接定义标线
  • 通过车道边界隐式表示
  • 依赖仿真引擎渲染标线
  • 无法自定义标线样式
查看详细属性 →

kawasaki.xodr

使用 roadMark 元素详细定义道路标线。
关键属性:
  • sOffset: 沿道路的起始位置
  • type: 标线类型 (none, solid, broken等)
  • weight: 线条粗细(standard, bold)
  • color: 标线颜色(standard, white, yellow等)
  • width: 标线宽度
  • laneChange: 是否允许变道
查看详细属性 →
