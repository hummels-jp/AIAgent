三种格式道路中心线特点概览

OSM (OpenStreetMap)

  • 使用 way 元素表示道路
  • WGS84 地理坐标系
  • 丰富的元数据和标签
  • 适合地图编辑和显示
  • 节点序列定义道路形状

SUMO (kawasaki.net.xml)

  • 使用 edge 元素表示道路
  • UTM 投影坐标系
  • 包含车道和速度信息
  • 适合交通流仿真
  • 明确的起止节点连接

OpenDRIVE (kawasaki.xodr)

  • 使用 road 元素表示道路
  • 高精度几何曲线
  • 支持复杂道路几何
  • 适合驾驶仿真
  • 支持道路连接关系

🗺️ OSM 格式 - way 元素

在 OpenStreetMap 中,道路中心线使用 <way> 元素表示,通过引用一系列节点 (<nd>) 来定义道路的几何形状。way 元素本身不存储坐标,坐标存储在被引用的 node 元素中。

way 元素属性

属性名 类型 说明
id Long 唯一的道路标识符 ,在整个 OSM 数据集中必须唯一
visible Boolean 是否可见,值为 "true" 或 "false"。通常为 "true"
version Integer 版本号,每次编辑后递增
changeset Long 最后一次编辑的变更集ID
timestamp DateTime 最后一次编辑的时间戳 (ISO 8601 格式)
user String 最后一次编辑的用户名
uid Integer 用户的唯一标识符

道路相关标签 (Tags)

标签键
标签值示例
 说明
highway primary, residential, tertiary, service, footway 道路类型 ,定义道路的等级和用途
name String 道路名称
name:en String 英文名称
name:ja String 日文名称
name:ja_rm String 日文罗马音
name:ja-Latn String 日文拉丁化名称
lanes Integer
车道数量
maxspeed Integer/String 最高限速,单位 km/h (如: 40)
oneway yes/no 是否为单向道路
ref String 道路编号或参考号
surface paved, asphalt, gravel 路面类型
cycleway lane, track, share_busway 自行车道类型
source String 数据来源 (如: Bing 2012)

🚗 SUMO 格式 - edge 元素

在 SUMO 中,道路中心线使用 <edge> 元素表示。edge 表示有向的连接,从起始节点到终止节点。双向道路用两个相反方向的 edge 表示(正ID和负ID)。

edge 元素属性

属性名 类型 说明
id String 边的唯一标识符 。负号表示相反方向(如: 124964556#0 / -124964556#0)
from String 起始节点的ID
to String 终止节点的ID
priority Integer 道路优先级,值越大优先级越高(1-15)
type String 道路类型 ,引用 type 定义(如: highway.unclassified, highway.primary)
shape Coordinates
道路中心线的几何形状,用空格分隔的坐标点(x,y)序列
function String
边的功能:normal(普通) 或 internal(路口内部连接)

lane 子元素 属性

属性名 类型 说明
id String 车道的唯一标识符 ,格式为 "edgeID_laneIndex"
index Integer
车道索引号,从0开始
speed Float
车道限速,单位 m/s (如: 13.89 = 50km/h)
length Float
车道长度,单位米
width Float
车道宽度,单位米(可选)
shape Coordinates
车道中心线的几何形状坐标
allow String 允许通行的车辆类型 ,用空格分隔(如: pedestrian delivery bicycle)
disallow String 禁止通行的车辆类型 ,用空格分隔

原始数据示例

示例说明:

🛣️
OpenDRIVE 格式 - road 元素

在 OpenDRIVE 中,道路中心线使用 <road> 元素表示。OpenDRIVE 提供最高精度的道路几何描述,支持直线、圆弧、螺旋线等复杂曲线,符合驾驶仿真 需求。

road 元素属性

属性名 类型 说明
name String 道路名称
length Float 道路的总长度,单位米
id Integer 道路的唯一标识符
junction Integer
所属路口的ID。-1表示不属于任何路口

road 子元素

子元素 说明
link 定义道路连接关系 (predecessor/successor)
type 道路类型 (town, motorway, rural等)
planView 平面视图,包含道路的几何曲线
elevationProfile
高程剖面,描述道路的高度变化
lateralProfile 横向剖面,描述道路的横坡
lanes
车道配置
,包含车道定义和标线
objects 道路上的对象(如标志牌、障碍物等)
signals 交通信号

geometry 元素属性 (planView 子元素 )

属性名 类型 说明
s Float 该几何段的起始位置,单位米
x Float 起始点的X坐标(UTM)
y Float 起始点的Y坐标(UTM)
hdg Float 起始航向角(heading),弧度制
length Float 该几何段的长度,单位米
