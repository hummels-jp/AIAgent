📚 OSM/SUMO/OpenDRIVE - Frequently Asked Questions

What values can the Highway attribute of OSM Links take, and what do they mean?

In OSM (OpenStreetMap), the highway attribute is used to describe the type and classification of roads. Below are common highway attribute values and their meanings:

highway value Meaning Description
motorway Motorway/Highway Highest class of road, typically with multiple lanes, median strips, and strict access control
motorway_link Motorway link Ramps or access roads connecting motorways to other roads
trunk Trunk road Important main roads, similar to motorways but may not be fully separated
trunk_link Trunk link Roads connecting to trunk roads
primary Primary road Important regional roads connecting major cities and regions
primary_link Primary link Roads connecting to primary roads
secondary Secondary road Roads connecting towns and villages
secondary_link Secondary link Roads connecting to secondary roads
tertiary Tertiary road Local roads connecting small towns and villages
residential Residential road Roads within residential areas, typically with lower speed limits
service Service road Service roads leading to parking lots, commercial areas, etc.
unclassified Unclassified road Smaller roads that don't fit into other classifications
living_street Living street Streets where pedestrians have priority, vehicles must slow down
pedestrian Pedestrian Roads restricted to pedestrian traffic only
track Track Unpaved roads for agricultural or forestry use
path Path Small paths for pedestrians or bicycles only
cycleway Cycleway Roads designated for bicycles only
busway Busway Roads designated for buses only
steps Steps Stairs or steps, generally unsuitable for vehicles
footway Footway Sidewalks for pedestrians only
bridleway Bridleway Roads usable by horses
Note: The highway attribute is not only used for roads but also includes non-motorized roads like footpaths and cycleways. When converting to SUMO or OpenDRIVE, different highway values will be mapped to different road types and attributes.
How to add satellite imagery corresponding to OSM data in QGIS

There are several ways to add satellite imagery corresponding to OSM data in QGIS. Here are some common methods:

Method 1: Using XYZ Tiles (Recommended)

  1. Open QGIS, find the "XYZ Tiles" layer in the left panel (usually at the bottom)
  2. Right-click on "XYZ Tiles" and select "New Connection..."
  3. In the pop-up window, enter:
    • Name: Give the connection a name (e.g., Google Satellite, Bing Maps, etc.)
    • URL: Enter the satellite imagery service URL (see common services below)
  4. Click "OK" to save the connection
  5. Double-click the connection you just created, and the satellite imagery will load into the canvas
  6. Ensure the satellite imagery layer is below the OSM data layer (drag to adjust layer order)

Common Satellite Imagery Service URLs:

Service Name URL Features
Google Satellite https://mt1.google.com/vt/lyrs=s&x={x}&y={y}&z={z} High-resolution satellite imagery, global coverage
Google Hybrid https://mt1.google.com/vt/lyrs=y&x={x}&y={y}&z={z} Satellite imagery + road labels
Bing Maps Satellite https://ecn.t3.tiles.virtualearth.net/tiles/a{q}.jpeg?g=1 Microsoft Bing satellite imagery
Esri Satellite https://server.arcgisonline.com/ArcGIS/rest/services/World_Imagery/MapServer/tile/{z}/{y}/{x} Esri global imagery
OpenStreetMap Standard https://tile.openstreetmap.org/{z}/{x}/{y}.png OSM standard map (not satellite)
CartoDB Dark Matter https://a.basemaps.cartocdn.com/dark_all/{z}/{x}/{y}@2x.png Dark base map (suitable for data visualization)

Method 2: Using QGIS Plugins

  1. Select Plugins → Manage and Install Plugins... from the menu bar
  2. Search for and install one of the following plugins:
    • QuickMapServices - Quickly add various map services
    • QuickOSM - Plugin specifically for OSM data
  3. After installation, find the plugin in the menu bar or toolbar and select the desired satellite imagery service

Method 3: Using WMS/WMTS Services

  1. Select Layer → Add Layer → Add WMS/WMTS Layer... from the menu bar
  2. Click "New" to create a new connection
  3. Enter the WMS service URL (e.g., ArcGIS online services, etc.)
  4. Select the required layers and add them to the map

Practical Tips:

  • Adjust Transparency: Double-click the satellite imagery layer and adjust transparency in the "Transparency" tab to easily view OSM data
  • Set Projection: Ensure all layers use the same coordinate system (WGS 84 / Pseudo-Mercator, EPSG:3857 is recommended)
  • Cache Management: You can manage cache location in Settings → Options → Map Options
  • Bookmarks: Use View → New Bookmark to save frequently used locations for quick navigation
Note: Using satellite imagery services requires compliance with their respective terms of service. Some services may require API keys or have usage restrictions. Google Maps services may require proxy access in certain regions.
Which highway attribute values in OSM data should be kept and which should be deleted

When converting OSM data to SUMO or OpenDRIVE, you need to choose whether to keep or delete certain highway attribute values based on your use case. Below are classification recommendations for common attribute values:

📊 Keep vs Delete Classification Table

highway value Recommendation Reason Use Case
NULL Delete Data missing or not filled, unable to determine road type All scenarios
footway Optional Pedestrian-only, vehicles cannot pass (unless footway analysis is needed) Pedestrian simulation, accessibility analysis
primary Keep Urban arterial roads, important traffic thoroughfares All scenarios
residential Keep Residential area roads, important part of urban traffic network Urban simulation
service Optional Service roads (parking lots, gas station entrances, etc.), usually not part of main road network Parking lot analysis, detailed micro-simulation
tertiary Keep Tertiary roads, connecting important nodes as branch roads Urban, regional simulation
unclassified Keep Unclassified public roads, typically small but passable roads Scenarios requiring complete road network
secondary Keep Secondary roads, connecting important towns All scenarios
motorway Keep Motorways, highest class roads All scenarios
motorway_link Keep Motorway links (ramps), necessary for entering/exiting motorways All scenarios
trunk Keep Trunk roads, similar to motorways but with less access control All scenarios
living_street Optional Living streets, pedestrian priority, very low speed Detailed residential area simulation
pedestrian Delete Pedestrian streets, vehicles cannot pass Vehicle simulation scenarios
path Delete Paths, for pedestrians or bicycles only Vehicle simulation scenarios
track Delete Unpaved roads, agricultural or forestry use Urban simulation scenarios
cycleway Delete Bicycle lanes, for bicycles only Vehicle simulation scenarios
steps Delete Steps, vehicles cannot pass All vehicle scenarios
busway Optional Bus-only lanes Bus simulation

🎯 Filtering Recommendations by Use Case

Scenario 1: Urban Road Traffic Simulation (Most Common)

Keep:motorway, motorway_link, trunk, primary, secondary, tertiary, residential, unclassified

Delete:NULL, pedestrian, footway, path, track, cycleway, steps, busway, living_street

Optional:service (can delete if parking lots and other detailed facilities are not needed)

Scenario 2: Motorway Network Simulation

Keep:motorway, motorway_link, trunk, trunk_link, primary

Delete:All other types

Scenario 3: Detailed Micro-simulation (Including Branch Roads)

Keep:motorway, motorway_link, trunk, primary, primary_link, secondary, secondary_link, tertiary, residential, unclassified, service

Delete:NULL, pedestrian, path, track, cycleway, steps

Optional:footway, living_street, busway

Scenario 4: Multi-modal Traffic Simulation (Vehicle + Pedestrian + Bicycle)

Keep:All passable road types (except NULL)

Delete:NULL

💡 Processing Recommendations

Recommended Processing Flow:
  1. Data Cleaning: First delete NULL values, this is invalid data that must be removed
  2. Scenario Matching: Select appropriate road type combinations based on simulation goals
  3. Connectivity Check: Check road network connectivity after deleting roads to avoid isolated breakpoints
  4. Attribute Mapping: Correctly map retained OSM highway values to SUMO/OpenDRIVE road types

🔧 How to Delete Unwanted Road Types in SUMO

When using the netconvert tool, you can use --remove-edges or --keep-edges parameters:

# Keep only specified types
netconvert --osm-files input.osm --output-file output.net.xml --keep-edges.by-type highway.primary,highway.secondary,highway.tertiary,highway.residential

# Delete specified types
netconvert --osm-files input.osm --output-file output.net.xml --remove-edges.by-type highway.footway,highway.path,highway.track
Special Note:Based on your detection list, it is recommended to keep primary, residential, tertiary, unclassified, delete NULL, and service and footway are optional based on needs. These four retained types already cover the main parts of the urban road network.
What are the road types in SUMO? What are the road types in OpenDRIVE style (Type)?

SUMO and OpenDRIVE each have their own road type systems. Below is a detailed comparison and explanation:

📊 SUMO Road Types (Type)

SUMO uses the "type" attribute to define road types, containing information such as speed, number of lanes, and priority. Common SUMO road types include:

Type Name Description Speed Limit (km/h) Lane Count Priority
motorway Motorway 130 2-4 15
motorway_link Motorway link 80 1-2 13
trunk Trunk road 100 2-3 12
trunk_link Trunk link 60 1-2 11
primary Primary road 80 2 10
primary_link Primary link 50 1-2 9
secondary Secondary road 60 1-2 8
secondary_link Secondary link 40 1 7
tertiary Tertiary road 50 1-2 6
residential Residential road 30 1-2 5
unclassified Unclassified road 30 1 4
service Service road 20 1 3
pedestrian Pedestrian street 10 1 1
living_street Living street 15 1 2

🛣️ OpenDRIVE Road Types (Type)

OpenDRIVE uses the `type` element to define road types, containing road rules, speed limits, and other information. OpenDRIVE standard defines the following road types:

Type Value Description Common Use Corresponding SUMO Type
unknown Unknown type Default type, not clearly classified unclassified
rural Rural road Roads in rural areas tertiary, secondary
motorway Motorway Motorway, restricted access motorway
town Town road Urban roads primary, secondary
lowSpeed Low speed road Roads with lower speed limits residential, service
pedestrian Pedestrian area Areas for pedestrian traffic only pedestrian
townExpressway Town expressway Urban expressways primary, trunk
townCollector Town collector road Roads connecting roads and destinations secondary, tertiary
townArterial Town arterial road Main urban traffic arteries primary, trunk
townPrivate Town private road Private residential area roads service, residential
townLocal Town local road Local roads residential, unclassified
town pedestrian Town pedestrian way Pedestrian areas in towns pedestrian

<types>
  <type id="motorway" speed="36.11" numberLanes="3" priority="15">
    <lane id="motorway.0" index="0" speed="36.11" length="0" shape=""/>
    <lane id="motorway.1" index="1" speed="36.11" length="0" shape=""/>
    <lane id="motorway.2" index="2" speed="36.11" length="0" shape=""/>
  </type>
  
  <type id="primary" speed="22.22" numberLanes="2" priority="10">
    <lane id="primary.0" index="0" speed="22.22" length="0" shape=""/>
    <lane id="primary.1" index="1" speed="22.22" length="0" shape=""/>

📐 OpenDRIVE Road Type XML Example

<road name="MainStreet" length="500.00" id="1" junction="-1">
  <link>
    <predecessor elementType="junction" elementId="100"/>
    <successor elementType="junction" elementId="101"/>
  </link>
  
  <type s="0.00" type="town" country="JPN">
    <speed max="13.89" unit="m/s"/>  <!-- 50 km/h -->
    <rule speedMax="13.89"/>
  </type>
  
  <planView>
    <geometry s="0.00" x="0.00" y="0.00" hdg="0.00" length="500.00">
      <line/>
    </geometry>
  </planView>
  
  <lanes>
    <laneSection s="0.00">
      <center>
        <lane id="0" type="driving" level="true">
          <width s="0.00" a="3.50"/>
        </lane>
      </center>
      <right>
        <lane id="-1" type="driving" level="true">
          <width s="0.00" a="3.50"/>
        </lane>
        <lane id="-2" type="driving" level="true">
          <width s="0.00" a="3.50"/>
        </lane>
      </right>
    </laneSection>
  </lanes>
</road>

⚙️ Custom Road Types

Custom Types in SUMO

<type id="my_custom_type" speed="15.00" numberLanes="2" priority="7"
      allow="passenger car taxi bus truck">
  <restriction speed="15.00"/>
</type>

Custom Types in OpenDRIVE

OpenDRIVE allows using any string as a type value, but it's recommended to use standard types:

<type s="0.00" type="custom_highway" country="JPN">
  <speed max="33.33" unit="m/s"/>  <!-- 120 km/h -->
  <rule speedMax="33.33"/>
</type>

📝 Key Differences Explanation

Feature SUMO OpenDRIVE
Type Definition Centrally defined in <types> element Defined separately in each <road> element
Speed Unit m/s m/s or km/h
Lane Definition Defined in type Defined in <lanes> element
Priority Higher numerical value means higher priority Implicitly defined by type
Vehicle Restrictions allow/disallow attributes Lane type attributes
Extensibility Supports custom types Supports arbitrary type strings
Note:When converting from SUMO to OpenDRIVE, SUMO road types will be mapped to OpenDRIVE standard types. If precise control over mapping relationships is needed, you can use SUMO's type file or manually edit the generated OpenDRIVE file.
