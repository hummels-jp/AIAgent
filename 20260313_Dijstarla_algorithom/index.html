📖 算法简介 / Algorithm Introduction / アルゴリズム紹介

Dijkstra 算法（中文名：迪杰斯特拉算法）是由荷兰计算机科学家艾兹赫尔·戴克斯特拉（Edsger W. Dijkstra）在1956年提出的一种图搜索算法。
Dijkstra's Algorithm is a graph search algorithm proposed by Dutch computer scientist Edsger W. Dijkstra in 1956.
Dijkstra（ダイクストラ）アルゴリズムは、オランダの計算機科学者エドガー・ダイクストラ（Edsger W. Dijkstra）が1956年に提案したグラフ探索アルゴリズムです。
该算法用于解决单源最短路径问题，即在给定一个带权重的有向图或无向图中，找到从给定起点到其他所有顶点的最短路径。Dijkstra 算法是图论中最经典、应用最广泛的算法之一。
This algorithm is used to solve the single-source shortest path problem, which is to find the shortest path from a given starting point to all other vertices in a weighted directed or undirected graph. Dijkstra's algorithm is one of the most classic and widely used algorithms in graph theory.
このアルゴリズムは単一始点最短経路問題を解くために使用されます。つまり、重み付き有向グラフまたは無向グラフにおいて、指定された始点から他のすべての頂点への最短経路を見つけることです。Dijkstraアルゴリズムはグラフ理論における最も古典的で広く使用されているアルゴリズムの1つです。

核心步骤 / Core Steps / 主要な手順

1. 初始化：将起点到自己的距离设为 0，到其他所有顶点的距离设为无穷大（∞）。
1. Initialization: Set the distance from the start point to itself as 0, and the distance to all other vertices as infinity (∞).
1. 初期化：始点から自分自身への距離を0に、他のすべての頂点への距離を無限大（∞）に設定します。
2. 选择当前顶点：从未被访问过的顶点中，选择距离起点最近的顶点作为当前顶点。
2. Select Current Vertex: From the unvisited vertices, select the vertex closest to the start point as the current vertex.
2. 現在の頂点を選択：未訪問の頂点の中から、始点に最も近い頂点を現在の頂点として選択します。
3. 更新邻居距离：遍历当前顶点的所有邻居，如果通过当前顶点到邻居的距离小于之前记录的距离，则更新邻居的距离和前驱节点。
3. Update Neighbor Distances: Traverse all neighbors of the current vertex. If the distance to a neighbor through the current vertex is less than the previously recorded distance, update the neighbor's distance and predecessor node.
3. 隣接ノードの距離を更新：現在の頂点のすべての隣接ノードを走査し、現在の頂点を通って隣接ノードに到達する距離が以前に記録された距離より小さい場合、その距離と前駆ノードを更新します。
4. 标记已访问：将当前顶点标记为已访问，之后不再作为当前顶点考虑。
4. Mark as Visited: Mark the current vertex as visited; it will no longer be considered as a current vertex.
4. 訪問済みとしてマーク：現在の頂点を訪問済みとしてマークし、以降は現在の頂点として考慮されません。
5. 重复步骤：重复步骤 2-4，直到所有顶点都被访问，或找到目标顶点。
5. Repeat Steps: Repeat steps 2-4 until all vertices have been visited or the target vertex is found.
5. 手順を繰り返す：すべての頂点が訪問されるか、ターゲット頂点が見つかるまで、手順2-4を繰り返します。

为什么有效？/ Why Effective? / なぜ有効なのか？

贪心选择：每次都选择距离最近的顶点，确保已经找到的路径是最优的。
Greedy Selection: Always choose the vertex with the closest distance, ensuring that the path found is optimal.
貪欲な選択：常に最も近い距離の頂点を選択することで、見つけられた経路が最適であることを保証します。
最优子结构：从起点到某个顶点的最短路径，也包含了路径中间所有顶点的最短路径。
Optimal Substructure: The shortest path from the start point to a vertex also includes the shortest paths to all intermediate vertices on that path.
最適部分構造：始点からある頂点への最短経路には、その経路上のすべての中間頂点への最短経路も含まれています。
无后效性：一旦顶点被标记为已访问，它的最短距离就不会再改变。
No Aftereffect: Once a vertex is marked as visited, its shortest distance will never change again.
後効果性なし：頂点が訪問済みとしてマークされると、その最短距離は二度と変わりません。

重要限制 / Important Limitations / 重要な制限

非负权重：Dijkstra 算法要求所有边的权重必须是非负的。如果存在负权边，算法可能得到错误结果。
Non-negative Weights: Dijkstra's algorithm requires that all edge weights must be non-negative. If there are negative weight edges, the algorithm may produce incorrect results.
非負の重み：Dijkstraアルゴリズムは、すべてのエッジの重みが非負であることを要求します。負の重みを持つエッジが存在する場合、アルゴリズムは間違った結果を生じる可能性があります。
单源最短路：Dijkstra 只能计算从一个起点到其他所有顶点的最短路径，不能直接计算任意两点之间的最短路径。
Single-source Shortest Path: Dijkstra can only calculate the shortest paths from one starting point to all other vertices, not directly calculate the shortest path between any two points.
単一始点最短経路：Dijkstraは1つの始点から他のすべての頂点への最短経路のみを計算でき、任意の2点間の最短経路を直接計算することはできません。
